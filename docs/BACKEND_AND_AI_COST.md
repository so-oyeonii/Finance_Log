# Smart Ledger Pro — 백엔드 구조 & AI 비용 분석

## 1. 서버/백엔드 구조

### 아키텍처 요약

```
Local-First 아키텍처 (서버 없이도 100% 동작)

┌───────────────────────────────────┐
│         브라우저 (클라이언트)         │
│                                   │
│  React/Next.js ←→ Dexie.js       │
│  (UI + 로직)      (IndexedDB)     │
│                                   │
│  모든 데이터 로컬 저장               │
│  네트워크 없이 완전 동작             │
└──────────┬────────────────────────┘
           │ AI 기능 사용 시에만
           │ fetch() POST 요청
           ▼
┌───────────────────────────────────┐
│   Next.js API Routes (서버리스)    │
│   /api/ai/smart-input             │
│   /api/ai/receipt                 │
│   /api/ai/chat                    │
│   /api/ai/analyze                 │
│   /api/ai/tts                     │
│                                   │
│   OPENAI_API_KEY (서버 전용)       │
└──────────┬────────────────────────┘
           │
           ▼
┌───────────────────────────────────┐
│         OpenAI API                │
│   gpt-4o-mini / tts-1            │
└───────────────────────────────────┘
```

### 핵심 포인트

| 항목 | 내용 |
|------|------|
| **프레임워크** | Next.js 14 (App Router) + TypeScript |
| **서버 유형** | 서버리스 (Vercel 배포 시 자동 스케일링) |
| **데이터 저장** | 100% 클라이언트 (IndexedDB via Dexie.js) |
| **서버 DB** | 없음 (Supabase 동기화는 Phase 7 예정) |
| **API 키 보호** | `OPENAI_API_KEY`는 서버 전용, 클라이언트 노출 없음 |
| **인증/보안** | 현재 미구현 (프로덕션 배포 시 추가 필요) |
| **미들웨어** | 없음 (Rate Limiting, Auth 미구현) |

### 서버가 하는 일 = AI 프록시 역할만

이 앱은 **서버가 거의 없는 구조**입니다:
- 데이터 CRUD는 전부 브라우저 IndexedDB에서 처리
- 서버(API Routes)는 오직 **OpenAI API 호출을 중계**하는 역할
- API 키를 클라이언트에 노출하지 않기 위한 프록시

## 3. AI 비용 분석

### OpenAI 모델별 가격표

| 모델 | Input (1M 토큰) | Output (1M 토큰) | 비고 |
|------|-----------------|-------------------|------|
| **gpt-4o-mini** | **$0.15** | **$0.60** | 현재 사용 중 |
| gpt-4o | $2.50 | $10.00 | 참고 (16.7배 비쌈) |
| gpt-4.1 | $2.00 | $8.00 | 최신 모델 |
| tts-1 | - | - | $15 / 1M 글자 |
| tts-1-hd | - | - | $30 / 1M 글자 |

> **gpt-4o-mini는 gpt-4o 대비 약 16.7배 저렴**

### 토큰 참고
- 한국어 1글자 ≈ 2~3 토큰
- 영어 1단어 ≈ 1~2 토큰
- 1,000 한국어 글자 ≈ 약 2,500 토큰

### 기능별 예상 비용 (gpt-4o-mini 기준)

## 4. 만약 gpt-4o로 교체한다면?

| 시나리오 | gpt-4o-mini | gpt-4o | 증가율 |
|----------|-------------|--------|--------|
| 가벼운 사용 | ~60원/월 | ~1,000원/월 | 16.7x |
| 보통 사용 | ~260원/월 | ~4,300원/월 | 16.7x |
| 헤비 사용 | ~680원/월 | ~11,000원/월 | 16.7x |

**권장사항:**
- 텍스트 기능 (파싱, 채팅, 분석): **gpt-4o-mini 유지** (충분한 성능)
- 영수증 OCR (Vision): 대량 사용 시 **gpt-4o 검토** (이미지 토큰 비용이 더 저렴)
- TTS: **tts-1 유지** (HD 불필요, nova 음성 한국어 양호)

## 5. 프로덕션 배포 시 추가 필요 사항

현재 미구현이지만, 배포 전 추가해야 할 것:

1. **Rate Limiting** — API 남용 방지 (IP당 분당 10회 등)
2. **인증** — 사용자별 API 접근 제어
3. **사용량 모니터링** — OpenAI 대시보드 + 자체 로깅
4. **비용 상한 설정** — OpenAI Usage Limits 설정
5. **에러 핸들링 강화** — 재시도 로직, 사용자 친화적 에러 메시지
6. **Supabase 동기화** — 선택적 클라우드 백업 (Phase 7)
